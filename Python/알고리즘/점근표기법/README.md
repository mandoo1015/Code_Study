# 점근표기법(Big-O)

## 시간 vs 공간

알고리즘을 따질 때 무엇이 더 중요한가? -> **시간**

- 공간은 재활용이 가능하지만, 시간은 재활용이 불가능하다.
- 공간은 확장이 가능하지만, 시간은 불가능하다.
- 100mb를 차지하는 0.1s 실행시간의 프로그램 vs 10mb를 차지하는 1s 실행시간의 프로그램 : 전자를 선택한다.

## 시간복잡도
입력의 크기가 증가함에 따라 시간이 얼마나 증가하는지를 나타내는 지표

①
| 입력 | 시간 |
| :----: | :----: |
| 1 | 10 |
| 2 | 20 |
| 5 | 50 |

②
| 입력 | 시간 |
| :----: | :----: | 
| 1 | 0.2 |
| 2 | 10 |
| 5 | 100 |

둘 중에 시간 복잡도가 더 낮은 것은? (더 빠른 알고리즘은?) -> ②번

2초에서 5초로 갈 때 : ①번 2.5배, ②번 10배 -> 입력이 증가함에 따라 시간이 가파르게 증가하면 느린 알고리즘이다.

## 점근표기법

계수나 상수와 상관없이 데이터 크기를 극단으로 늘렸을 때, **최고차항이 무엇인지**만 시간에 영향을 미친다.

O(n²) : 입력크기가 n배면, 시간은 n²배

입력크기가 커질수록 시간이 적게 늘어나는 알고리즘이 좋은 알고리즘

제일 좋은 알고리즘 : **상수 시간 알고리즘**

O(1) : 입력크기를 아무리 늘려도, 실행시간은 1초다.
