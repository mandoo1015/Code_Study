# 탐색과 정렬

## 탐색

- **선형 탐색** : 처음부터 끝까지 하나하나씩 비교해서 찾기
- **이진 탐색** : 반으로 나누어서 탐색

### 무엇이 더 좋은 알고리즘인가?
**최악**을 따져봐야 한다.

**64개의 리스트가 있을 때**
- 선형 탐색 최악의 경우 : `64`회
- 이진 탐색 최악의 경우 : `6`회

**128개의 리스트가 있을 때**
- 선형 탐색 최악의 경우 : `128`회
- 이진 탐색 최악의 경우 : `7`회

**이진 탐색**이 더 효율적인 알고리즘이다.

### 그렇다면 선형 탐색은 쓸모없는 알고리즘인가?
그건 아니다.

이진 탐색은 효율적이지만 조건이 있다. -> **리스트가 정렬되어 있어야 한다.**

선형 탐색은 정렬되어 있지 않은 리스트에도 사용 가능하다.

## 정렬

**선택 정렬**
- `i`(0부터 데이터의 끝)의 인덱스를 최솟값의 인덱스로 지정 `min_idx`
- `j`(i + 1부터 데이터의 끝)를 이동시키며 `min_idx`의 값과 비교
- `min_idx`의 값보다 `j`의 값이 더 작다면 `min_idx`위치 갱신
- `min_idx`의 값과 `j`의 값 바꾸기
- 반복

**삽입 정렬**
- `i`(1부터 데이터의 끝)의 값은 `key`
- `j`(i부터 0까지 역순)와 `i`를 비교
- `key`보다 `j`의 값이 작다면 `j` 오른쪽으로 이동
- `key`보다 `j`의 값이 크거나 같다면 그 위치에 삽입
- 반복
