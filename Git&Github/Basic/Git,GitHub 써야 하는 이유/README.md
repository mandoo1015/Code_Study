### 코드 관리, 이렇게 하고 있었다.

Git을 배우기 전에 나는 이렇게 파일을 관리했다.
```
프로젝트_최종.zip
프로젝트_최종_진짜.zip
프로젝트_최종_진짜_마지막.zip
프로젝트_최종_진짜_마지막_수정.zip
```
교수님이 "저번 버전으로 돌려봐"라고 하면?? 끝이다. 어떤 파일이 저번 버전인지 알 수가 없다.

---

### Git이 필요한 이유
Git은 **버전 관리 시스템**이다. 파일의 변경 이력을 자동으로 기록해준다.

**(1) 변경 이력이 남는다.**
Git은 파일의 변경사항을 커밋 단위로 저장한다. 언제, 누가, 뭘 바꿨는지 다 남는다.
```bash
git log
```
"지난주에 뭘 고쳤더라?"가 아니라 그냥 기록을 보면 된다.

**(2) 실수해도 되돌릴 수 있다.**
코드를 잘못 수정했다? 예전 버전으로 돌아가면 된다.
```bash
git reset --hard
```
최종\_진짜\_마지막" 파일을 찾는 것보다는 훨씬 나을 것이다

**(3) 여러 기능을 동시에 개발할 수 있다**
브랜치 기능으로 메인 코드를 건드리지 않고 새 기능을 개발할 수 있다. 완성되면 합치고, 실패하면 그냥 버리면 된다.

---

### GitHub가 필요한 이유
Git이 내 컴퓨터에서 버전 관리를 해준다면, GitHub는 그걸 인터넷에 올려두는 곳이다.

**1. 백업**
노트북이 고장나도 GitHub에 코드가 있으면 살 수 있다. 이거 하나만으로도 쓸 이유가 충분하다.

**2. 협업**
팀 프로젝트할 때 GitHub로 같은 저장소를 공유하면 누가 뭘 수정했는지 충돌없이 합칠 수 있다.

**3. 포트폴리오**
개발자 취업할 때 GitHub 프로필은 거의 필수다. 내가 뭘 만들었는지 보여줄 수 있는 가장 좋은 방법이다.

---

### 정리
Git : 내 컴퓨터에서 코드 변경 이력을 관리
GitHub : Git으로 관리한 코드를 인터넷에 저장하고 공유
